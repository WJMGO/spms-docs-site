# SPMS 仪表板设计 - 管理层月底绩效处理版本

## 📋 文档概述

本文档针对管理层月底处理绩效数据、调整排名、进行部门间排序的使用场景，重新设计仪表板。核心目标是**突出部门成员的排序分布，便于实时调整和查看**。

---

## 🎯 使用场景分析

### 业务流程

```
月底 (25-26号)
    ↓
收集所有评分数据
    ↓
管理层打开仪表板
    ↓
按部门查看员工排序
    ↓
调整个别员工的分数/排名
    ↓
查看调整后的排序分布
    ↓
部门间排序穿插（跨部门对标）
    ↓
确认最终排序
    ↓
定版并发布
    ↓
月度总结会议上公示
```

### 关键需求

1. **快速查看排序** - 清晰展示部门内员工的排序
2. **实时调整** - 修改分数后立即看到排序变化
3. **排序分布** - 可视化展示分数分布（如：优秀、良好、一般等）
4. **部门对比** - 便于部门间排序穿插和对标
5. **批量操作** - 支持批量调整和导出

## 📊 核心功能详解

### 1. 快速筛选区域

**部门筛选**：
```
[全部部门] [技术部] [产品部] [运营部] [财务部]
```
- 点击部门快速切换
- 显示该部门的所有员工排序

**排序方式**：
```
[排序方式] ▼
- 按总分降序（默认）
- 按总分升序
- 按名字排序
- 按职位排序
- 自定义排序
```

**显示范围**：
```
[显示范围] ▼
- 显示所有
- 显示前 10 名
- 显示前 20 名
- 显示优秀（90-100分）
- 显示良好（80-89分）
- 显示一般（70-79分）
- 显示及格（60-69分）
```

---

### 2. 员工排序表

**表格列**：
| 列 | 说明 | 功能 |
| :--- | :--- | :--- |
| **排名** | 部门内排名（1-N） | 显示排名，可拖拽调整 |
| **姓名** | 员工名称 | 点击查看详情 |
| **职位** | 员工职位 | 辅助信息 |
| **总分** | 最终评分 | 可编辑，修改后自动重新排序 |
| **等级** | 绩效等级 | 根据分数自动判断 |
| **操作** | 快速操作 | 编辑、调整排名等 |

**交互功能**：

1. **编辑分数**
   - 点击总分单元格，弹出编辑框
   - 输入新分数，按 Enter 确认
   - 系统自动重新排序
   - 显示排序变化提示

2. **调整排名**
   - 点击"调整排名"按钮
   - 弹出排名调整对话框
   - 可以手动输入新排名
   - 或者拖拽调整排名

3. **查看详情**
   - 点击员工名称
   - 显示该员工的详细评分信息
   - 显示各评分项的得分
   - 支持查看评分历史

## 🔄 排序调整流程

### 场景 1：修改单个员工的分数

**步骤**：
1. 在排序表中找到员工
2. 点击总分单元格
3. 输入新分数，按 Enter
4. 系统自动重新排序
5. 排序分布自动更新
6. 显示排名变化提示（如："张三排名从第 3 名升至第 2 名"）

**示例**：
```
修改前：张三 总分 88.7，排名第 3
修改后：张三 总分 92.3，排名第 2
提示：排名上升 1 位 ↑
```

### 场景 2：调整排名（跨部门排序穿插）

**步骤**：
1. 打开"部门对比"视图
2. 查看各部门的排序分布
3. 识别需要调整的员工
4. 点击员工的"调整排名"按钮
5. 在对话框中输入新排名
6. 系统自动调整其他员工的排名
7. 排序分布自动更新

**示例**：
```
技术部张三（总分 92.3）当前排名第 2
产品部李四（总分 92.0）当前排名第 1
管理层决定：张三应该排在李四前面
操作：调整张三的排名为第 1
结果：张三排名第 1，李四排名第 2（同部门内）
```

### 场景 3：批量调整

**步骤**：
1. 选择多个员工（勾选复选框）
2. 点击"批量调整"按钮
3. 选择调整方式：
   - 统一加分/减分
   - 统一调整等级
   - 统一调整排名
4. 输入调整参数
5. 预览调整结果
6. 确认调整

**示例**：
```
选择技术部的 5 个员工
批量调整：统一加分 2 分
原分数：88, 87, 86, 85, 84
新分数：90, 89, 88, 87, 86
排序自动更新
```

## 🔄 数据更新流程

### 实时更新

当用户修改分数时：
1. 系统立即更新排序
2. 排序分布自动重新计算
3. 部门对比自动更新
4. 显示排名变化提示
5. 记录修改历史

**API 调用**：
```
PATCH /api/trpc/assessment.updateScore
  - assessmentId: 评分 ID
  - newScore: 新分数
  
返回：
  - updatedAssessment: 更新后的评分
  - newRank: 新排名
  - oldRank: 旧排名
  - distributionStats: 更新后的分布统计
```

## 💡 设计建议

### 1. 实时反馈
- 修改分数后立即显示排名变化
- 显示排名上升/下降的提示
- 排序分布图表实时更新

### 2. 撤销/重做
- 支持撤销最后一次修改
- 支持重做已撤销的修改
- 显示修改历史

### 3. 数据验证
- 分数范围验证（0-100）
- 排名范围验证
- 防止重复排名

### 4. 性能优化
- 大数据量时使用虚拟滚动
- 排序计算使用后端处理
- 缓存部门对比数据

### 5. 安全性
- 定版后禁止修改
- 记录所有修改操作
- 支持修改审计

这个设计方案针对管理层月底处理绩效数据的实际需求，突出了：

2. **实时调整** - 修改分数立即看到排序变化
3. **分布统计** - 可视化的排序分布
4. **部门对比** - 便于部门间排序穿插
