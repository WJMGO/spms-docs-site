# SPMS 仪表板设计 - 自定义评分周期版本

## 📋 文档概述

本文档基于自定义评分周期的需求，重新设计 SPMS 仪表板。评分周期通常为上月26号到本月25号，但可根据节假日和年底调整。

---

## 🎯 核心需求分析

### 评分周期特点

**标准周期**：
- 开始日期：上月 26 号
- 结束日期：本月 25 号
- 周期长度：约 30 天

**特殊情况**：
- 春节前提前核算
- 年底提前核算
- 其他大型节假日调整

**管理需求**：
- 支持创建自定义周期
- 支持修改周期时间
- 支持周期状态管理（进行中、已结束、已发布）
- 支持多个周期并存

## 📊 关键指标（基于选中周期）

所有的关键指标都是基于**选中的评分周期**计算的。

### 1. 总评分数

**含义**：选中周期内的所有评分数量

**计算**：
```
总评分数 = 该周期内所有员工的评分数
         = 所有状态的评分之和
```

**示例**：
- 2026年3月周期：156 份评分
- 2026年2月周期：148 份评分
- 2026年1月周期：152 份评分

**业务含义**：
- 反映该周期的评分规模
- 用于评估工作量
- 支持周期间对比

---

### 2. 已完成

**含义**：该周期内已完成的评分数（已提交 + 已批准）

**计算**：
```
已完成 = 该周期内 (已提交 + 已批准) 的评分数
```

**示例**：
- 2026年3月周期：98 份已完成
- 2026年2月周期：145 份已完成（周期已结束）
- 2026年1月周期：150 份已完成（周期已结束）

**业务含义**：
- 反映该周期的完成进度
- 用于评估是否按时完成
- 支持进度预警

---

### 3. 待处理

**含义**：该周期内还需处理的评分（草稿 + 待审批）

**计算**：
```
待处理 = 该周期内 (草稿 + 待审批) 的评分数
```

**示例**：
- 2026年3月周期：35 份待处理
- 2026年2月周期：3 份待处理（周期即将结束）
- 2026年1月周期：2 份待处理（周期已结束，可能是补充评分）

**业务含义**：
- 反映还需完成的工作
- 用于识别瓶颈
- 支持优先级管理

---

### 4. 平均分

**含义**：该周期内所有已批准评分的平均分

**计算**：
```
平均分 = 该周期内所有已批准评分的总分 / 已批准评分数
```

**示例**：
- 2026年3月周期：82.5 分（周期进行中）
- 2026年2月周期：81.2 分（周期已结束）
- 2026年1月周期：83.8 分（周期已结束）

**业务含义**：
- 反映该周期的整体绩效
- 用于周期间对比
- 支持绩效改进评估

---

## 📈 评分进度展示

### 完成率

**含义**：该周期的评分完成进度

**计算**：
```
完成率 = 已完成 / 总评分数 × 100%
       = 98 / 156 × 100%
       = 62.8%
```

**进度等级**：
```
0-25%：刚刚开始
25-50%：进行中
50-75%：接近完成
75-100%：即将结束或已结束
```

### 周期倒计时

**显示**：剩余时间（如果周期进行中）

**示例**：
- 进行中：剩余 5 天
- 已结束：已于 2026-03-25 结束
- 已发布：已于 2026-03-26 发布结果

**表单字段**：

- 开始日期
- 结束日期

- 是否为标准周期

```
周期名称：2026年4月评分周期
开始日期：2026-03-26
结束日期：2026-04-25
描述：标准月度评分周期
标准周期：是
```

### 编辑周期

**可修改字段**：
- 周期名称
- 开始日期（如果周期未开始）
- 结束日期（如果周期未结束）
- 周期描述

**限制**：
- 已结束的周期不能修改时间
- 已发布的周期不能修改

### 查看历史周期

**显示列表**：
```

2026年2月评分周期    2026-01-26   2026-02-25   已结束    [查看]
2026年1月评分周期    2025-12-26   2026-01-25   已发布    [查看]
```

## 🔄 周期状态流转

```
创建 → 进行中 → 已结束 → 已发布
       ↓
     编辑（仅在进行中时）
```

### 状态说明

| 状态 | 说明 | 操作 |
| :--- | :--- | :--- |
| **进行中** | 周期已开始，可以提交评分 | 可编辑、可提交评分 |
| **已结束** | 周期已过期，不能再提交新评分 | 可查看、可批准评分 |
| **已发布** | 周期已发布结果，不能修改 | 仅可查看 |

## 💡 设计建议

### 1. 周期切换的影响

当用户切换评分周期时，以下内容应该更新：
- ✅ 关键指标（总评分数、已完成、待处理、平均分）
- ✅ 完成率进度条
- ✅ 最近评分列表
- ✅ 部门统计数据
- ✅ 所有报表数据

### 2. 默认显示周期

- 优先显示"进行中"的周期
- 如果没有"进行中"的周期，显示最新的周期
- 用户可以手动切换到其他周期

### 3. 周期提示

- 如果周期即将结束（< 3 天），显示红色警告
- 如果周期已结束，显示灰色提示
- 如果有多个"进行中"的周期，显示选择提示

### 4. 响应式设计

**桌面版**：
- 周期选择器在顶部
- 完整显示所有信息

**平板版**：
- 周期选择器可能需要折叠
- 关键指标可能需要两行显示

**手机版**：
- 周期选择器为下拉菜单
- 关键指标单列显示

**功能**：在当前周期内创建新的绩效评分

**跳转页面**：
- `/assessment/create?periodId={currentPeriodId}` - 新建评分页面

**功能**：查看当前周期的各类报表

**跳转页面**：

## 📊 数据流示例

### 场景：用户切换周期

**步骤**：
1. 用户打开仪表板，看到"2026年3月评分周期"
2. 用户点击周期选择器的下拉菜单
3. 用户选择"2026年2月评分周期"
4. 系统更新所有数据：
   - 关键指标更新为 2 月周期的数据
   - 进度条更新为 2 月周期的完成率
   - 最近评分列表更新为 2 月周期的评分
   - 部门统计更新为 2 月周期的统计

**API 调用**：
```
GET /api/trpc/dashboard.getMetrics?periodId=202602
GET /api/trpc/dashboard.getRecentAssessments?periodId=202602
GET /api/trpc/dashboard.getDepartmentStats?periodId=202602
```
