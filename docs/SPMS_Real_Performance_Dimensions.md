## 📊 绩效评分体系总览

### 总分构成

| 维度 | 满分 | 权重 | 说明 |
|------|------|------|------|
| **日常工作** | 100 分 | 50% | 项目成效、解决问题数量和难度 |
| **工作质量** | 15 分 | 7.5% | 代码走查、代码审核、Bug 打回率、设计评审等 |
| **个人目标** | 15 分 | 7.5% | 关键指标和关键事项的达成情况 |
| **部门互评** | 5 分 | 2.5% | 相关方对团队交付、协作、态度的评价 |
| **绩效加分** | 15 分 | 7.5% | 团队贡献、分享、专利、文档、表扬等 |
| **绩效减分** | - | - | 技术失误、管理失误、质量异常等 |
| **总分** | **150 分** | **100%** | 最终绩效评分 |

## 📋 详细维度说明

### 1. 日常工作（100 分）

**评分标准**：根据解决的项目问题数量及难度

**操作办法**：
- 细则请参考各组考核标准及 Jira 评估细则
- 问题 Jira 数量以"实际完成日期"和"开发工程师"两个字段为准

**特殊规则**：
1. 功能或需求开发部分研发测试通过前，此部分的低级缺陷等级为 NA 或降级
2. 鼓励提升开发质量：功能开发过程中带出的 bug 视情况降低权重

**计算方式**：
- 基于 Jira 数据自动计算
- 考虑问题的难度和复杂度
- 鼓励高质量的开发

---

### 2. 工作质量（15 分）

**评分标准**：代码质量、代码审核、Bug 打回率、超期问题、设计评审

#### 2.1 代码走查（0-4 分）
- **组织者**：1-2 分/场
- **参与者**：0.5 分/场（有效建议）

#### 2.2 代码审核（0-3 分）
- **基础评分**：0.5 分/次（审核出异常加分）
  - Error：1 分
  - 其他：0.5 分
- **额外加分**：
  - 代码审核数 ≥30 个：+1 分
  - 代码审核数 ≥50 个：+2 分
- **上限**：3 分

#### 2.3 部门 Bug 打回率（0-2 分）
- **≤3%**：2 分/人
- **≤5%**：1 分/人
- **>5%**：0 分/人

#### 2.4 个人 Bug 有效打回（-1 分/个）
- 每个有效打回的 Bug：-1 分
- 相同类型的问题算 1 个

#### 2.5 超期问题部门达成（3 分/人）
- 与质量组达成一致的目标：3 分/人

#### 2.6 设计评审（0-4 分）
- **组织者**：1-2 分/场
- **参与者**：0.5 分/场（有效建议）

**总分上限**：15 分

---

### 3. 个人目标（15 分）

**评分标准**：关键指标和关键事项的达成情况

**流程**：
1. 工程师制定 OAR（目标、行动、结果）
2. 部门经理审核并给出目标分
3. 部门公示
4. 执行与监控
5. 工程师总结达成情况
6. 部门经理评定最终得分
7. 最终结果部门公示

#### 3.1 关键指标（0-7 分）
- **月度 Sprint 平均完成率**：
  - 90%-95%：1 分
  - 95%-100%：2 分
- **月度项目里程碑达成率 100%**：0-5 分（按难度评估）

#### 3.2 关键事项（0-8 分）
- **部门 TopN 目标**：0-8 分
- **CBB 或功能元目标**：0-8 分
- **动态的改善和创新目标**：0-8 分
- 需部门内全员共识
- 按难度评估出预估得分

**难度评估标准**（以 T14 职级为参照）：
- **1-3 分**（常规任务，1-3 天）：本部门流程/质量/专项改善、功能元迭代与新增
- **4-6 分**（挑战任务，4-6 天）：跨部门流程/质量/专项改善、CBB 迭代与新增、新技术攻关、复杂架构设计
- **7-8 分**（突破性任务，7-10 天）：技术创新、解决行业级难题

**最终得分计算**：
- 最终得分 = 目标分 × 完成度
- 完成度统计月度增量值

---

### 4. 部门互评（5 分）

**评分标准**：相关方对团队交付、协作、态度的评价

**评价维度**：
1. **团队交付**（按时、质量等维度）
2. **团队协作**（问题解答、技术协助等）
3. **态度和反馈**

**操作办法**：
- 每月 25 号之前完成调查问卷评分反馈以及统计

**评分规则**：
- **前 20%**：5 分
- **前 50%**：3 分
- **后 30%**：1 分

---

### 5. 绩效加分（15 分）

**总分上限**：15 分

#### 5.1 团队贡献 - 培养新人（0-3 分）
- **第 1 月**：0-3 分
- **第 2 月**：0-2 分
- **第 3 月**：0-1 分
- **第 4 月**：无

**规则**：
- 实习生与校招入职新人是同一位，只做一次加分
- 导师自己及部门团队成员每月根据该导师带新人的成效，确认是否加满该月的分数
- 实习考核工作量导师自行登记 Jira 跟进，视情况登记

#### 5.2 分享目标（0-4 分）
- **1-2 分/次**
- **上限**：≤4 分

#### 5.3 专利目标（0-6 分）
- **交底书登记 Jira**：提案通过 3 分，交局通过 3 分
- **上限**：≤6 分

#### 5.4 文档目标（0-2 分）
- **1 分/份**
- **上限**：≤2 分

#### 5.5 团队氛围（0-2 分）
- **1-2 分/次**（活动组织等）
- **上限**：≤2 分

#### 5.6 微创新（0-3 分）
- **根据项目结项成效**：1-3 分
- **上限**：≤3 分

#### 5.7 招聘面试（0-3 分）
- **0.5 分/人**
- **上限**：≤3 分（超出 6 人部分按日常工作量统计）

#### 5.8 表扬（相关方）（0-8 分）
- **点赞平台等非正式表扬**：1 分（上限≤3 分）
- **个人书面邮件正式表扬**：5 分
- **项目组成员邮件正式表扬**：2 分

#### 5.9 其他 - 项目 PL（0-5 分）
- **S 级、A 级项目 PL**：增加问卷调查
- **问卷对象**：项目经理 + 部门经理 + 该项目的系统开发成员
- **问卷维度**：
  - 主导跟进：5 分
  - 沟通协调：5 分
  - 专业能力：5 分
- **评分规则**：
  - 根据相关方反馈情况、项目按时达成情况及项目等级
  - 大 PL：加 0-5 分
  - 小 PL：加 0-3 分

---

### 6. 绩效减分

**总分清零条件**：当月清 0

#### 6.1 技术失误 / 管理失误

| 金额范围 | 严重程度 | 扣分 |
|---------|---------|------|
| 1 万以下 | 一般 | 2-5 分 |
| 1 万-5 万 | 严重 | 5-10 分 |
| 5 万-10 万 | 很严重 | 10-20 分 |
| 10 万以上 | 极其严重 | 当月清 0 |

#### 6.2 低级失误

| 金额范围 | 严重程度 | 扣分 |
|---------|---------|------|
| 5K 以下 | 一般 | 5-10 分 |
| 5K-3 万 | 严重 | 10-30 分 |
| 3 万以上 | 极其严重 | 当月清 0 |

#### 6.3 质量异常

| 异常类型 | 扣分 |
|---------|------|
| 软件送测异常 | 4 分/次 |
| Jenkins 编译错误 | 1 分/次 |
| 代码审核导致的质量异常事件 | 占失误分数的 30% |

## 🔄 绩效评分流程

### 完整流程

```
1. 工程师制定 OAR
   ↓
2. 部门经理审核并给出目标分
   ↓
3. 部门公示
   ↓
4. 执行与监控
   ↓
5. 项目变化/紧急插入变更调整（可选）
   ↓
6. 工程师总结达成情况
   ↓
7. 部门经理评定最终得分
   ↓
8. 最终结果部门公示
```

### 变更调整流程

**触发条件**：
- 项目方向调整
- 需求大幅变更
- 关键资源缺失
- 出现更高优先级的紧急任务（事件、售后问题或关键在研项目攻关等）

**处理步骤**：
1. 发起变更（工程师或项目经理或部门经理）
2. 说明变更原因、原目标调整建议、新目标提案
3. 部门经理对变更申请进行审核
4. 评估新目标的难度和价值
5. 重新给出目标分
6. 再次公示

**时间要求**：一天内此流程闭环

## 📊 绩效等级划分

根据总分（满分 150 分）：

| 等级 | 分数范围 | 说明 |
|------|---------|------|
| **优秀** | 90-100 | 表现突出，超额完成目标 |
| **良好** | 80-89 | 表现良好，完成目标 |
| **一般** | 70-79 | 表现一般，基本完成目标 |
| **及格** | 60-69 | 表现一般，勉强完成目标 |
| **不及格** | <60 | 表现不佳，未完成目标 |

## 💡 关键特点

### 1. **多维度评估**
- 不仅看产出（日常工作），也看过程（工作质量、个人目标）
- 不仅看个人表现，也看团队贡献（部门互评、绩效加分）

### 2. **激励机制**
- 绩效加分项丰富，鼓励分享、创新、团队建设
- 培养新人有额外加分
- 表扬有额外加分

### 3. **约束机制**
- 绩效减分项明确，对失误有明确的处罚
- 严重失误可导致当月清 0

### 4. **过程管理**
- 强调目标设定和监控
- 支持目标变更调整
- 透明的评分过程

### 5. **公平性**
- 所有评分都要部门公示
- 支持异议期
- 评分标准明确

## 🎯 系统实现建议

### 1. 数据库表设计

```sql
-- 绩效维度表
CREATE TABLE performance_dimensions (
  id INT PRIMARY KEY,
  name VARCHAR(50),           -- 日常工作、工作质量、个人目标等
  max_score DECIMAL(5,2),     -- 满分
  weight DECIMAL(5,2),        -- 权重
  description TEXT
);

-- 绩效评分表
CREATE TABLE performance_scores (
  id INT PRIMARY KEY,
  employee_id INT,
  period_id INT,
  daily_work_score DECIMAL(5,2),      -- 日常工作
  work_quality_score DECIMAL(5,2),    -- 工作质量
  personal_goal_score DECIMAL(5,2),   -- 个人目标
  department_review_score DECIMAL(5,2), -- 部门互评
  bonus_score DECIMAL(5,2),           -- 绩效加分
  penalty_score DECIMAL(5,2),         -- 绩效减分
  total_score DECIMAL(5,2),           -- 总分
  status VARCHAR(20),                 -- 草稿、已提交、已批准
  created_at TIMESTAMP,
  updated_at TIMESTAMP
);

-- 工作质量详情表
CREATE TABLE work_quality_details (
  id INT PRIMARY KEY,
  score_id INT,
  code_review_count INT,
  code_review_score DECIMAL(5,2),
  code_audit_count INT,
  code_audit_score DECIMAL(5,2),
  bug_return_rate DECIMAL(5,2),
  bug_return_score DECIMAL(5,2),
  personal_bug_count INT,
  personal_bug_score DECIMAL(5,2),
  design_review_count INT,
  design_review_score DECIMAL(5,2),
  overdue_problem_score DECIMAL(5,2),
  total DECIMAL(5,2)
);

-- 绩效加分详情表
CREATE TABLE bonus_details (
  id INT PRIMARY KEY,
  score_id INT,
  new_person_training_score DECIMAL(5,2),
  sharing_score DECIMAL(5,2),
  patent_score DECIMAL(5,2),
  document_score DECIMAL(5,2),
  innovation_score DECIMAL(5,2),
  team_atmosphere_score DECIMAL(5,2),
  recruitment_score DECIMAL(5,2),
  praise_score DECIMAL(5,2),
  pl_score DECIMAL(5,2),
  total DECIMAL(5,2)
);

-- 绩效减分详情表
CREATE TABLE penalty_details (
  id INT PRIMARY KEY,
  score_id INT,
  technical_error_amount DECIMAL(10,2),
  technical_error_score DECIMAL(5,2),
  management_error_amount DECIMAL(10,2),
  management_error_score DECIMAL(5,2),
  low_error_amount DECIMAL(10,2),
  low_error_score DECIMAL(5,2),
  quality_anomaly_count INT,
  quality_anomaly_score DECIMAL(5,2),
  total DECIMAL(5,2)
);
```

### 2. 前端表单设计

**第一步**：基本信息
- 选择员工
- 选择评分周期
- 选择评分模板

**第二步**：各维度评分
- 日常工作：Jira 数据自动导入
- 工作质量：各子项输入
- 个人目标：关键指标和关键事项
- 部门互评：问卷调查结果
- 绩效加分：各子项输入
- 绩效减分：异常情况输入

**第三步**：总结和提交
- 显示各维度得分
- 显示总分和等级
- 显示排名
- 提交评分

### 3. 管理层仪表板功能

- **员工排序表**：按总分排序，支持部门筛选
- **维度分布**：显示各维度的分数分布
- **部门对比**：跨部门的绩效对比
- **排名穿插**：支持在部门间穿插排名
- **分数调整**：支持点击编辑分数
- **批量调整**：支持批量调整（如统一加分）
- **定版发布**：确认无误后定版并发布
