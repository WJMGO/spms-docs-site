## 🔢 关键指标详解

### 1. 总评分数 (Total Assessments)

**显示值**：156

**含义**：
- 本评分周期内，系统中所有员工的绩效评分总数
- 包括所有状态的评分：草稿、已提交、已批准、已拒绝

**计算公式**：
```
总评分数 = 草稿数 + 已提交数 + 已批准数 + 已拒绝数
         = 23 + 35 + 98 + 0
         = 156
```

**业务含义**：
- 反映本周期的评分规模
- 用于了解评分工作量
- 支持评分计划和资源分配

**示例场景**：
- 公司有 150 名员工，本周期应该有 150 份评分
- 如果显示 156，说明有 6 份是补充或重新评分

---

### 2. 已完成 (Completed Assessments)

**显示值**：98

**含义**：
- 已经完成评分流程的评分数量
- 包括两种状态：
  - **已提交**（Submitted）- 评分者已提交，等待批准
  - **已批准**（Approved）- 管理员已批准，流程完成

**计算公式**：
```
已完成 = 已提交数 + 已批准数
       = 35 + 63
       = 98
```

**业务含义**：
- 反映评分工作的进度
- 用于评估是否按时完成评分
- 支持进度跟踪和预警

**状态流转**：
```
草稿 → 提交 → 批准 → 完成
      ↓
    拒绝 → 返回编辑
```

**示例场景**：
- 本周期有 150 份评分，已完成 98 份
- 完成率 = 98 / 150 = 65.3%
- 说明还有 52 份评分未完成

---

### 3. 待处理 (Pending Assessments)

**显示值**：35

**含义**：
- 需要进一步处理的评分数量
- 包括两种状态：
  - **草稿**（Draft）- 评分者正在编辑，未提交
  - **待审批**（Awaiting Approval）- 已提交但未批准

**计算公式**：
```
待处理 = 草稿数 + 待审批数
       = 23 + 12
       = 35
```

**业务含义**：
- 反映还需要完成的工作
- 用于识别瓶颈和风险
- 支持优先级管理

**处理流程**：
```
草稿状态：
- 评分者需要继续编辑并提交
- 可以保存为草稿，稍后继续编辑

待审批状态：
- 管理员需要审核并批准或拒绝
- 如果拒绝，返回给评分者重新编辑
```

**示例场景**：
- 张三的评分还在草稿状态，需要完成编辑
- 李四的评分已提交，等待经理批准
- 这两份都属于"待处理"

---

### 4. 平均分 (Average Score)

**显示值**：82.5

**含义**：
- 本周期所有已批准评分的平均分数
- 只计算已完成（已批准）的评分
- 范围：0-100 分

**计算公式**：
```
平均分 = 所有已批准评分的总分 / 已批准评分数量
       = (85.5 + 90 + 78.5 + ... + 80) / 63
       = 82.5
```

**业务含义**：
- 反映本周期的整体绩效水平
- 用于评估团队表现
- 支持绩效对标和改进

**分数等级参考**：
```
90-100：优秀（Excellent）
80-89：良好（Good）
70-79：一般（Fair）
60-69：及格（Pass）
0-59：不及格（Fail）
```

**示例场景**：
- 本周期平均分 82.5，属于"良好"等级
- 去年同期平均分 80，说明今年有所提升
- 可以用来评估绩效改进效果

## 📈 评分进度展示

### 完成率进度条

**显示**：62.8%

**含义**：
- 已完成评分数 / 总评分数
- 用进度条可视化展示

**计算公式**：
```
完成率 = 已完成 / 总评分数 × 100%
       = 98 / 156 × 100%
       = 62.8%
```

**业务含义**：
- 快速了解评分工作的整体进度
- 用于评估是否按计划进行
- 支持进度预警和调整

**进度等级**：
```
0-25%：刚刚开始
25-50%：进行中
50-75%：接近完成
75-100%：即将结束
```

### 已批准 vs 待处理

**已批准**：98
- 绿色卡片
- 表示工作已完成，可以结案

**待处理**：35
- 黄色卡片
- 表示还需要继续跟进

## 🎨 系统功能模块

### 1. 📋 评分管理 (Assessment Management)

**功能**：
- 创建新的绩效评分
- 编辑已有的评分
- 提交评分
- 查看评分历史

**用户**：
- 评分者（经理、主管）
- 被评分者（员工）

**流程**：
```
创建评分 → 填写评分项 → 保存草稿 → 提交 → 等待批准 → 批准/拒绝
```

---

### 2. 📊 数据分析 (Data Analysis)

**功能**：
- 统计分析评分数据
- 生成各类报表
- 导出数据为 CSV/Excel
- 趋势分析和对标

**用户**：
- 管理员
- HR
- 部门负责人

**报表类型**：
- 公司整体绩效报表
- 部门绩效报表
- 员工排名报表
- 评分项统计报表
- 趋势对比报表

---

### 3. 👥 员工管理 (Employee Management)

**功能**：
- 维护员工信息
- 管理部门和职位
- 设置评分权限
- 查看员工档案

**用户**：
- HR
- 系统管理员

**管理内容**：
- 员工基本信息
- 部门关系
- 职位信息
- 权限设置

---

### 4. 📈 趋势分析 (Trend Analysis)

**功能**：
- 分析评分趋势
- 对比历史数据
- 预测未来趋势
- 生成改进建议

**用户**：
- 管理层
- HR

**分析维度**：
- 时间维度 - 不同周期的对比
- 部门维度 - 部门间的对比
- 员工维度 - 个人的成长轨迹
- 项目维度 - 不同评分项的表现

## 💡 设计建议

### 1. 颜色编码
- 🟢 **绿色** - 已完成、进行顺利
- 🟡 **黄色** - 待处理、需要关注
- 🔴 **红色** - 已拒绝、需要改进
- 🔵 **蓝色** - 已提交、等待处理

### 2. 图标使用
- 📋 - 评分相关
- 📊 - 数据分析
- 👥 - 员工管理
- 📈 - 趋势分析
- ✓ - 已完成
- ⏱ - 待处理

### 3. 信息层级
- **一级** - 关键指标（总评分数、已完成、待处理、平均分）

## 🎓 使用场景示例

### 场景 1：管理员晨会

**时间**：每天上午 9:00

**操作**：
1. 打开仪表板，查看"概览"标签页
2. 检查完成率是否按计划进行
3. 查看"最近评分"标签页，了解最新活动
4. 如果有拒绝的评分，点击查看详情

**决策**：
- 如果完成率低于预期，需要提醒相关人员
- 如果有拒绝的评分，需要跟进改进

---

### 场景 2：HR 周报

**时间**：每周五下午

**操作**：
1. 打开仪表板，查看"部门统计"标签页
2. 对比各部门的完成率和平均分
3. 点击"查看报表"生成详细报表
4. 导出数据为 Excel，准备周报

**决策**：
- 识别进度落后的部门
- 分析绩效差异的原因
- 制定改进措施

---

### 场景 3：管理层月度汇报

**时间**：每月月末

**操作**：
1. 打开仪表板，记录本月的关键指标
2. 点击"查看报表"，生成月度绩效报表
3. 对比上月数据，分析趋势
4. 导出图表，准备汇报材料

**决策**：
- 评估本月绩效目标完成情况
- 识别优秀部门和员工
- 制定下月改进计划

---

## 📋 总结

SPMS 仪表板通过关键指标、进度展示、数据统计等多个维度，为管理者提供了全面的绩效管理视图。通过清晰的信息架构和直观的可视化设计，支持快速决策和高效管理。
