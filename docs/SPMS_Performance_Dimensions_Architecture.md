# SPMS 绩效管理系统 - 基于维度的页面架构设计

## 📋 核心需求分析

### 用户角色
- **部门经理** - 查看和微调本部门员工绩效
- **管理员** - 查看全部部门员工绩效，进行跨部门调整

### 首页需求
- **全部部门的绩效总分排序**
- **展示各维度的分数**
- **支持部门筛选**
- **基于此页面支持各维度的微调**

---

## 🎯 绩效表格维度分析

根据常见的绩效评分体系，绩效表格通常包含以下维度：

### 1. **日常工作** (30%)
- 工作完成度
- 工作效率
- 工作态度
- 团队协作

### 2. **工作质量** (30%)
- 代码质量
- 交付质量
- 问题处理能力
- 持续改进

### 3. **创新能力** (15%)
- 技术创新
- 流程优化
- 知识分享
- 问题解决

### 4. **绩效加分** (15%)
- 重大贡献
- 技术突破
- 客户满意度
- 其他加分

### 5. **绩效扣分** (10%)
- 严重错误
- 违规行为
- 客户投诉
- 其他扣分

## 🔄 用户操作流程

### 场景 1：查看全部部门排序
```
1. 打开首页
2. 查看全部部门员工的总分排序
3. 可以看到每个员工的各维度分数
```

### 场景 2：调整某个维度的分数
```
1. 在首页上，点击某个维度的分数（如"日常工作"列）
2. 进入该维度的详情页面
3. 查看该维度下各个员工的分数
4. 点击编辑按钮，修改分数
5. 系统自动重新计算总分和排名
6. 返回首页查看更新后的排序
```

### 场景 3：筛选部门并调整
```
1. 在首页上选择部门（如"技术部"）
2. 查看该部门员工的排序
3. 点击某个维度进入详情页
4. 在该部门的维度详情页中调整分数
5. 返回首页查看更新后的排序
```

### 场景 4：跨部门对比和调整
```
1. 在首页上选择"全部部门"
2. 查看所有部门的排序
3. 对比不同部门的员工分数
4. 进入某个维度详情页
5. 跨部门调整分数以达到合理的排序
```

## 🚀 开发优先级

### Phase 1: 首页 + 基础维度页面
1. 首页 - 全部部门绩效排序（支持部门筛选、排序）
2. 日常工作维度详情页
3. 工作质量维度详情页
4. 分数编辑功能

### Phase 2: 其他维度页面
1. 创新能力维度详情页
2. 绩效加分维度详情页
3. 绩效扣分维度详情页

### Phase 3: 高级功能
1. 员工详情页
2. 批量调整功能
3. 历史版本查看
4. 定版发布功能

---

## 💡 关键设计原则

1. **维度优先** - 所有操作都围绕维度展开
2. **实时计算** - 修改任何分数都立即重新计算总分和排名
3. **可追溯** - 记录所有分数修改的历史
4. **跨部门对标** - 支持跨部门的分数对比和调整
5. **简洁高效** - 最少的点击次数完成调整

## 📌 与现有系统的关系

- **评分表单页面** - 用于员工提交绩效评分（保留）
- **评分列表页面** - 用于查看评分进度（保留）
- **管理层首页** - 替换为新的维度-based 首页
- **维度详情页** - 新增，用于维度级别的调整
